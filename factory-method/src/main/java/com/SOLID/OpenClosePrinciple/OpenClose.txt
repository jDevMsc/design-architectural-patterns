Принцип открытости - закрыточти

Классы открыты для расширения: означает, что поведение сущности может быть расширено путём создания новых типов сущностей
Классы закрыты для изменения: в результате расширения поведения сущности, не должны вноситься изменения в код, который эта сущность использует.

Система должна быть построена таким образом, что все ее последующие изменения должны быть реализованы с помощью добавления нового кода, а не изменения уже существующего.


==================Пример
Класс повар который (метод)готовит картошку. Если хотим добавить функционал, не должны измениять повара.
Например исп. паттерн стратегия для создания разных видов еды
Cook bob = new Cook();
bob.MakeDinner(new PotatoMeal());
bob.MakeDinner(new SaladMeal());
Закрыто на изменения, но расширяема с помощью интерфейса
